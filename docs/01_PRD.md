# 01 产品需求文档 PRD

## 1. 产品名称

暂定：Meeting Copilot Local

## 2. 产品定位

面向程序员、产品经理、销售、创业者和管理者的实时会议辅助工具。

核心价值：

```text
会议中实时理解上下文，帮助用户记录、判断、回应和跟进。
```

## 3. 目标用户

### 3.1 技术人员

典型场景：

- 需求评审；
- 架构评审；
- Bug 复盘；
- 上线评审；
- 跨团队沟通。

核心需求：

- 自动识别技术风险；
- 提醒边界条件；
- 提醒需要确认的接口、数据、异常、回滚问题；
- 会后沉淀待办。

### 3.2 销售/商务

典型场景：

- 客户需求沟通；
- 售前方案讨论；
- 报价谈判；
- 项目推进会。

核心需求：

- 抓客户痛点；
- 识别预算和决策链；
- 识别异议；
- 生成下一步跟进话术。

### 3.3 管理者/老板

典型场景：

- 项目会议；
- 周会；
- 复盘会；
- 经营分析会。

核心需求：

- 抓决策；
- 抓责任人；
- 抓风险；
- 抓待办；
- 防止会议空转。

## 4. 核心功能

### 4.1 会议监听

用户可以：

- 选择麦克风输入设备；
- 开启/关闭系统输出监听；
- 查看当前是否正在录音；
- 暂停/恢复监听；
- 结束会议。

验收标准：

- 开启监听后，App 能获取麦克风音频；
- 开启系统输出后，App 能获取电脑播放音频；
- UI 上必须显示明显的录音状态；
- 用户可以随时暂停。

### 4.2 实时转写

用户可以看到实时字幕流。

字幕字段：

```json
{
  "time": "00:03:12",
  "speaker": "Me | Remote | Speaker 1",
  "text": "这个需求如果下周上线，会不会影响支付流程？",
  "is_final": true
}
```

验收标准：

- 音频输入后 1～3 秒内出现文字；
- 临时结果和最终结果要有状态区分；
- 能按时间顺序展示；
- 能保存到本地。

### 4.3 说话人标记

MVP 先做到：

```text
Me：本地麦克风
Remote：系统输出混音
```

第二阶段做到：

```text
Speaker 1
Speaker 2
Speaker 3
```

限制说明：

- 混音后的远程音频不能天然知道真实姓名；
- 不得把 Speaker 1 自动显示成某个真实人名；
- 后续可以通过声纹注册或会议 SDK 增强。

### 4.4 实时问题识别

系统自动识别：

- 显性问题；
- 隐性风险；
- 异议；
- 决策点；
- 待办；
- 需求变更；
- 时间/成本/范围冲突；
- 用户痛点。

输出示例：

```json
{
  "event_type": "risk_question",
  "priority": "high",
  "evidence": "如果下周上线，会不会影响支付流程？",
  "reason": "涉及上线时间和支付链路稳定性"
}
```

### 4.5 大模型建议

当触发高价值事件时，系统生成建议。

建议内容包括：

1. 当前问题本质；
2. 风险点；
3. 建议回应；
4. 建议追问；
5. 是否形成待办。

UI 展示要简短，避免刷屏。

建议示例：

```text
问题本质：这是上线风险和支付链路稳定性的冲突。
建议回应：先确认支付链路是否改动，再讨论灰度范围。
可直接说：我们先拆成影响范围和回滚方案两个问题确认，避免一次性影响全部用户。
```

### 4.6 会后纪要

会议结束后生成：

- 会议摘要；
- 关键讨论；
- 决策；
- 待办；
- 风险；
- 未解决问题；
- 原始转写记录。

导出格式：

- Markdown；
- JSON；
- 后续支持 PDF/DOCX。

## 5. 非功能需求

### 5.1 实时性

- VAD 检测延迟：小于 300ms；
- ASR 首字延迟：目标 1～3 秒；
- 建议生成延迟：目标 3～8 秒；
- UI 更新：WebSocket 推送。

### 5.2 稳定性

- 音频设备断开时要提示；
- ASR 服务异常时 UI 不崩溃；
- LLM 调用失败时保留转写；
- 应支持会议中断恢复。

### 5.3 隐私

- 默认本地存储；
- 默认不上传音频；
- 云端 LLM 调用前提示用户；
- API Key 本地安全保存；
- 提供一键删除会议记录。

### 5.4 可扩展性

- ASR 可替换；
- LLM 可替换；
- Diarization 可替换；
- 音频采集可按平台扩展；
- 场景助手可通过 Prompt 模板扩展。

## 6. MVP 页面

### 6.1 首页

元素：

- 开始会议按钮；
- 选择音频设备；
- 选择会议模式；
- 选择助手角色；
- 隐私提示。

### 6.2 会议中页面

布局：

```text
左侧：实时转写时间轴
右侧：实时建议卡片
底部：开始/暂停/结束/导出
顶部：录音状态和音频设备状态
```

### 6.3 会后页面

内容：

- 会议摘要；
- 决策；
- 待办；
- 风险；
- 转写全文；
- 导出按钮。

## 7. 成功指标

MVP 验证指标：

- 能否稳定监听 30 分钟会议；
- 转写是否基本可读；
- 建议是否在关键问题出现后及时弹出；
- 用户是否愿意在下一次会议继续打开；
- 用户是否觉得建议比单纯纪要更有价值。
