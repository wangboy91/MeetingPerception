# 10 安全、隐私与合规设计

## 1. 基本原则

会议内容是高敏感数据。

默认策略：

```text
默认本地处理
默认不上传音频
默认不上传全文
默认不公开分享
用户主动授权才使用云端模型
```

## 2. 用户授权

开始会议前必须提示：

```text
请确认你有权记录当前会议内容，并已遵守所在地区、组织和会议参与方的相关要求。
```

UI 必须提供：

- 明显录音状态；
- 暂停按钮；
- 停止按钮；
- 删除记录按钮；
- 云端 LLM 开关。

## 3. 数据分类

| 数据 | 敏感等级 | 默认处理 |
|---|---|---|
| 原始音频 | 高 | 不长期保存 |
| 转写文本 | 高 | 本地保存，可关闭 |
| 建议内容 | 中高 | 本地保存 |
| 摘要纪要 | 高 | 本地保存 |
| 设备信息 | 中 | 本地保存 |
| API Key | 高 | 安全存储 |

## 4. 原始音频策略

MVP 建议：

```text
默认不保存原始音频
只保留实时处理所需的短暂 buffer
buffer 处理后释放
```

如果后续支持保存音频，必须：

- 用户显式开启；
- UI 明确显示；
- 支持单独删除；
- 支持加密存储。

## 5. 云端 LLM 策略

启用云端 LLM 时，不发送音频，只发送必要文本上下文。

发送内容：

```text
当前触发句
最近短期上下文
结构化会议状态
必要摘要
```

不发送：

```text
完整会议全文
原始音频
本地设备信息
用户 API Key
```

除非用户明确开启。

## 6. 日志策略

默认日志不能包含完整会议文本。

允许记录：

```text
session_id
模块状态
错误码
耗时
模型名称
调用是否成功
```

默认禁止记录：

```text
完整音频
完整转写
完整 prompt
完整 LLM response
API Key
```

debug 模式可以临时开启，但 UI 必须提示。

## 7. API Key 存储

优先使用系统安全存储：

- Windows Credential Manager；
- macOS Keychain；
- Linux Secret Service。

MVP 如果来不及：

- 本地配置文件加密；
- 明确标记为临时方案；
- 不要提交任何 key 到仓库。

## 8. 权限处理

需要权限：

- 麦克风权限；
- 系统音频采集权限；
- 本地文件读写；
- 网络访问，仅当启用云端模型。

权限失败时必须提示具体原因。

## 9. 删除机制

用户可以：

- 删除单场会议；
- 删除所有会议；
- 删除模型配置；
- 删除 API Key；
- 清空日志。

删除后不可在 UI 中恢复。

## 10. 风险提示

产品内置说明：

```text
本工具仅用于辅助记录和分析会议内容。使用者应确保对会议记录、转写和模型处理行为具有合法授权。
```

## 11. 企业版预留

后续企业版可增加：

- 本地私有化模型；
- 禁止云端 LLM；
- 统一审计；
- 管理员策略；
- 敏感词脱敏；
- 数据保留期限；
- 加密导出。

## 12. 验收标准

- 开始会议前有授权提示；
- 会议中有明显录音状态；
- 默认不保存原始音频；
- 默认不上传音频；
- 云端 LLM 可关闭；
- 日志不包含完整会议内容；
- 可删除会议记录。
